Splash Technologies

Product DesignerB2B SaaSDesign System
splash technologies
PROBLEM

Scaling design without a system

As the product grew, inconsistencies multiplied across features and teams. There was no shared language between design and engineering, which slowed development and created a fragmented user experience.

APPROACH

Build the foundation, then the features

I audited every screen and component, cataloged patterns, and identified gaps. Built a token-based design system with reusable Figma components, usage guidelines, and partnered with engineering for code implementation.

IMPACT

Faster shipping, fewer inconsistencies

The design system reduced handoff time significantly and brought visual consistency across the entire product. It became the single source of truth for both teams.

WHAT'S INSIDE

Token System

Color, type, and spacing tokens

Component Library

40+ reusable Figma components

Documentation

Usage guidelines for every pattern

Dev Handoff

Coded components matching 1:1

Created by Ben Sykes
PROBLEM

Design-to-code handoff is still broken

The gap between design and production has always created friction. I wanted to challenge that by building something real entirely within Figma Make.

APPROACH

Design and ship in the same tool

Built a fully functional typing speed test end-to-end using Figma Make. Real-time WPM tracking, accuracy scoring, daily rotating challenges — all in one environment.

IMPACT

A live proof of concept

TypeRace is live and publicly accessible. A designer shipping a production-quality product without traditional engineering handoff.

WHAT'S INSIDE

Real-Time WPM

Live speed calculated as you type

Accuracy Tracking

Error rate logged per session

Daily Challenges

Rotating prompts to keep it fresh

Visual Feedback

Instant correct/incorrect states
